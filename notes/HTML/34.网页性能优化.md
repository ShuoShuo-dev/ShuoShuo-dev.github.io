# 网页性能优化

## 一、为什么要优化性能

- **用户体验**：页面加载超过3秒，53%的用户会离开
- **搜索引擎排名**：Google将页面速度作为排名因素
- **转化率**：页面加载每慢1秒，转化率下降约7%
- **流量费用**：对移动用户尤为重要

---

## 二、关键渲染路径优化

### 2.1 关键渲染路径是什么

浏览器将HTML、CSS、JavaScript转化为屏幕像素的过程：

```
HTML → DOM → CSSOM → Render Tree → Layout → Paint → Composite
```

### 2.2 优化CSS

```html
<!-- 1. 将关键CSS内联 -->
<head>
    <style>
        /* 首屏需要的样式 */
        .header { background: #333; color: #fff; }
        .hero { min-height: 100vh; }
        /* 其他样式 */
    </style>

    <!-- 非关键CSS异步加载 -->
    <link rel="preload" href="non-critical.css" as="style"
          onload="this.onload=null;this.rel='stylesheet'">
    <noscript><link rel="stylesheet" href="non-critical.css"></noscript>
</head>
```

```html
<!-- 2. 使用 media 属性 -->
<link rel="stylesheet" href="print.css" media="print">
```

### 2.3 优化 JavaScript

```html
<!-- 1. 脚本放在底部 -->
<body>
    <main>...</main>
    <script src="app.js"></script>
</body>

<!-- 2. 使用 defer 属性 -->
<script defer src="app.js"></script>

<!-- 3. 使用 async 属性（非依赖脚本） -->
<script async src="analytics.js"></script>

<!-- 4. 内联关键脚本 -->
<script>
    // 首屏必需的JavaScript
</script>
```

## 三、资源加载优化

### 3.1 预加载关键资源

```html
<!-- 预加载字体 -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

<!-- 预加载关键图片 -->
<link rel="preload" as="image" href="hero.jpg">

<!-- 预加载关键CSS -->
<link rel="preload" as="style" href="critical.css">

<!-- 预加载脚本 -->
<link rel="preload" as="script" href="app.js">

<!-- 预连接 -->
<link rel="preconnect" href="https://api.example.com">
<link rel="dns-prefetch" href="https://api.example.com">
```

### 3.2 预获取和预渲染

```html
<!-- 预获取下一个导航需要的资源 -->
<link rel="prefetch" href="next-page.html">
<link rel="prefetch" href="next-page-data.json">

<!-- 预渲染整个页面 -->
<link rel="prerender" href="next-page.html">
```

### 3.3 字体优化

```html
<!-- 1. 使用 woff2 格式 -->
<link rel="preload" href="fonts/lato.woff2" as="font" type="font/woff2" crossorigin>

<!-- 2. 使用 font-display -->
<style>
@font-face {
    font-family: 'Lato';
    src: url('fonts/lato.woff2') format('woff2');
    font-display: swap; /* 可选: swap, block, optional */
}
</style>

<!-- 3. 只加载需要的字体 -->
<style>
/* 使用 unicode-range 加载特定字符 */
@font-face {
    font-family: 'IconFont';
    src: url('icons.woff2') format('woff2');
    unicode-range: U+00-FF; /* 只加载拉丁字符 */
}
</style>
```

## 四、图片优化

### 4.1 图片格式选择

| 场景 | 推荐格式 | 原因 |
|------|----------|------|
| 照片 | JPEG、WebP、AVIF | 体积小 |
| 透明图片 | PNG、WebP、AVIF | 支持透明通道 |
| 图标 | SVG | 矢量，可缩放 |
| 简单图形 | PNG、WebP | 色彩简单时体积小 |

### 4.2 图片懒加载

```html
<!-- 原生懒加载 -->
<img src="photo.jpg" loading="lazy" alt="...">

<!-- JavaScript 懒加载 -->
<img class="lazy" data-src="photo.jpg" alt="...">

<script>
const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            const img = entry.target;
            img.src = img.dataset.src;
            observer.unobserve(img);
        }
    });
});
document.querySelectorAll('.lazy').forEach(img => observer.observe(img));
</script>
```

### 4.3 响应式图片

```html
<img src="small.jpg"
     srcset="small.jpg 480w,
             medium.jpg 800w,
             large.jpg 1200w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="响应式图片">
```

### 4.4 图片尺寸

```html
<!-- 1. 明确宽高 -->
<img src="photo.jpg" width="800" height="600" alt="...">

<!-- 2. 使用 srcset 提供不同尺寸 -->
<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w"
     width="800"
     height="600"
     alt="...">

<!-- 3. 避免使用图片缩放 -->
<!-- ❌ 不好 -->
<img src="large.jpg" width="100"> <!-- 应该直接用小图 -->

<!-- ✅ 好 -->
<img src="small.jpg" width="100">
```

## 六、缓存策略

### 6.1 使用缓存

```html

<!-- Cache-Control: max-age=31536000, immutable -->

<!-- 2. 使用版本号 -->
<link rel="stylesheet" href="styles.v2.css">

<script>
if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js');
}
</script>
```
