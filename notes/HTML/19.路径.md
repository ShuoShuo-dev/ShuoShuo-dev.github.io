# 路径

## 什么是路径

路径是描述文件或资源位置的字符串，在HTML中用于指定图片、视频、CSS、JS等资源的地址。

## 绝对路径

绝对路径是指从根目录开始的完整路径。

### 网络绝对路径

```html
<img src="https://example.com/images/photo.jpg">
<link href="https://example.com/css/style.css">
<a href="https://example.com/about.html">关于我们</a>
```

### 本地绝对路径

```html
<!-- Windows -->
<img src="C:/Users/Admin/Documents/images/photo.jpg">

<!-- Linux/Mac -->
<img src="/home/user/images/photo.jpg">
```

### 绝对路径的特点

| 优点 | 缺点 |
|------|------|
| 位置明确，不会混淆 | 不够灵活，移植困难 |
| 任何地方都能访问 | 换了环境可能失效 |

## 相对路径

相对路径是相对于当前文件位置的路径。

### 当前目录

```html
<!-- 同一目录下 -->
<img src="photo.jpg">

<!-- 等价写法 -->
<img src="./photo.jpg">
```

### 子目录

```html
<!-- images目录下 -->
<img src="images/photo.jpg">

<!-- 嵌套子目录 -->
<img src="images/logos/company.png">
```

### 上级目录

```html
<!-- 上一级目录 -->
<img src="../photo.jpg">

<!-- 上两级目录 -->
<img src="../../photo.jpg">

<!-- 混合使用 -->
<img src="../../images/photo.jpg">
```

### 路径含义

| 符号 | 含义 |
|------|------|
| `/` | 根目录 |
| `./` | 当前目录（可省略） |
| `../` | 上一级目录 |
| `../../` | 上两级目录 |

## 相对路径示例

假设文件结构如下：

```
website/
├── index.html          （当前文件）
├── about.html
├── css/
│   └── style.css
├── images/
│   ├── logo.png
│   └── banner.jpg
├── js/
│   └── main.js
└── pages/
    └── contact.html
```

### 从index.html出发

```html
<!-- 当前目录 -->
<a href="about.html">关于我们</a>

<!-- 子目录 -->
<link href="css/style.css">
<img src="images/logo.png">
<script src="js/main.js"></script>

<!-- 上级目录（从pages出发时） -->
<img src="../images/banner.jpg">
```

### 从pages/contact.html出发

```html
<!-- 回到上级目录 -->
<a href="../index.html">首页</a>
<a href="../about.html">关于我们</a>

<!-- 访问上级目录的资源 -->
<link href="../css/style.css">
<img src="../images/logo.png">

<!-- 继续访问同级其他目录 -->
<img src="../images/banner.jpg">
```

## 根目录相对路径

以 `/` 开头，表示从网站根目录开始。

```html
<!-- 无论在哪个页面，都从根目录开始 -->
<img src="/images/logo.png">
<link href="/css/style.css">
<a href="/pages/contact.html">联系我们</a>
```

### 对比

| 路径类型 | 示例 | 说明 |
|----------|------|------|
| 相对路径 | `images/logo.png` | 相对于当前文件 |
| 根相对路径 | `/images/logo.png` | 相对于网站根目录 |
| 绝对路径 | `https://example.com/images/logo.png` | 完整URL |

## 路径选择建议

### 使用相对路径的情况

1. **站内资源链接**：推荐使用相对路径
2. **小型项目**：相对路径更灵活
3. **需要移植的项目**：相对路径便于迁移

```html
<!-- 推荐：站内使用相对路径 -->
<img src="./images/logo.png">
<link href="./css/style.css">
<a href="./about.html">关于我们</a>
```

### 使用绝对路径的情况

1. **外部资源**：引用其他网站的资源
2. **确定不变的URL**：如CDN资源

```html
<!-- 推荐：外部资源使用绝对路径 -->
<img src="https://cdn.example.com/library.js">
<link href="https://fonts.googleapis.com/css2?family=Roboto">
```

## 常见错误

### 1. 路径大小写

```html
<!-- Windows不区分大小写，但服务器区分 -->
<img src="Images/logo.png">    <!-- 可能找不到 -->
<img src="images/logo.png">    <!-- 正确 -->
```

### 2. 路径中的空格

```html
<!-- 空格可能导致问题 -->
<img src="my images/logo.png">

<!-- 解决方案：使用URL编码或删除空格 -->
<img src="my%20images/logo.png">
<img src="my-images/logo.png">
```

### 3. 缺少文件扩展名

```html
<!-- 缺少扩展名 -->
<img src="logo">              <!-- 错误 -->

<!-- 正确 -->
<img src="logo.png">
```

### 4. 路径末尾的斜杠

```html
<!-- 大多数情况下末尾斜杠可选 -->
<img src="images/logo.png">
<img src="images/logo.png/">

<!-- 但在某些服务器配置下可能有区别 -->
```

## 最佳实践

1. **保持目录结构清晰**：按照资源类型分类
2. **使用相对路径**：便于项目迁移
3. **统一命名规范**：小写字母+连字符
4. **注意路径大小写**：确保文件名正确
5. **避免中文路径**：可能产生编码问题

```html
<!-- 好的命名 -->
<img src="images/product-logo.png">
<img src="images/product_logo.png">

<!-- 避免 -->
<img src="images/产品LOGO.png">
```

## 总结

- **绝对路径**：从根目录或完整URL开始，适合外部资源
- **相对路径**：相对于当前文件，适合站内资源
- **根相对路径**：从网站根目录开始，适合大型项目
- 选择合适的路径类型，提高代码的可维护性
