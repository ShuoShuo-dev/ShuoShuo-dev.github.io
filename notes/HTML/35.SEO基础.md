# SEO基础

## 一、什么是SEO

SEO（Search Engine Optimization，搜索引擎优化）是通过优化网站结构、内容和技术细节，提高网站在搜索引擎结果页面（SERP）中排名的过程。

---

## 二、HTML与SEO的关系

### 2.1 搜索引擎工作原理

```
爬虫抓取 → 索引建立 → 排名计算 → 结果展示
```

### 2.2 HTML对SEO的影响

| HTML元素 | SEO作用 |
|----------|---------|
| `<title>` | 搜索结果标题显示 |
| `<meta description>` | 搜索结果摘要 |
| `<h1>`-`<h6>` | 内容层次结构 |
| `<a>` | 页面链接关系 |
| `<img alt>` | 图片索引 |
| `<header>` | 页面头部信息 |

## 三、Title 标签优化

### 3.1 Title 标签规范

```html
<!-- ✅ 好的Title -->
<title>Python基础教程 - 从入门到精通 | 编程学习网</title>

<!-- ❌ 不好的Title -->
<title>首页</title>
<title>欢迎来到我的网站</title>
<title>公司名 - 公司名 - 公司名</title>
```

### 3.2 Title 书写原则

- **长度**：50-60个字符内
- **包含关键词**：重要的词放前面
- **独特性**：每个页面有不同的Title
- **品牌标识**：可包含网站名称

```html
<!-- 推荐格式 -->
<title>页面主关键词 - 次要关键词 | 品牌名</title>
<title>页面主关键词 | 品牌名</title>
```

## 四、Meta 标签优化

### 4.1 Description

```html
<!-- 搜索结果摘要（通常150-160字符） -->
<meta name="description"
      content="学习Python编程的最佳在线教程，涵盖基础语法、数据结构、
               Web开发等，适合零基础入门和进阶提升。">

<!-- ❌ 不好的description -->
<meta name="description" content="编程教程">
<meta name="description"
      content="这是一个关于编程、代码、软件开发、JavaScript、Python、
               HTML、CSS、Java、数据库、网络、技术的综合性技术博客网站。">
```

### 4.2 Keywords（虽已不那么重要）

```html
<!-- 虽然搜索引擎不太看重，但仍可填写 -->
<meta name="keywords" content="Python教程, Python入门, Web开发">
```

### 4.3 Robots

```html
<!-- 允许搜索引擎索引和跟踪链接 -->
<meta name="robots" content="index, follow">

<!-- 不允许索引，但跟踪链接 -->
<meta name="robots" content="noindex, follow">

<!-- 完全禁止 -->
<meta name="robots" content="noindex, nofollow">
```

### 4.4 其他重要Meta

```html
<!-- 作者 -->
<meta name="author" content="张三">

<!-- 版权 -->
<meta name="copyright" content="©2024 编程学习网">

<!--  viewport - 移动端必需 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 字符编码 -->
<meta charset="UTF-8">

<!-- 语言 -->
<meta http-equiv="Content-Language" content="zh-CN">

<!-- 定时跳转（慎用） -->
<meta http-equiv="refresh" content="5;url=https://new-site.com">
```

## 五、语义化HTML结构

### 5.1 正确的标题层次

```html
<body>
    <header>
        <h1>网站名称</h1>
        <nav>导航</nav>
    </header>

    <main>
        <h1>页面主标题（每个页面只有一个）</h1>

        <article>
            <h2>文章章节1</h2>
            <h3>子章节1.1</h3>
            <h3>子章节1.2</h3>

            <h2>文章章节2</h2>
            <h3>子章节2.1</h3>
        </article>

        <aside>
            <h2>侧边栏标题</h2>
        </aside>
    </main>

    <footer>
        <h2>底部信息</h2>
    </footer>
</body>
```

:::warning
- 每个页面**只能有一个 `<h1>`**
- 标题层级**不能跳跃**（如 h2 → h4）
:::

### 5.2 语义化标签使用

```html
<!-- ✅ 语义化 -->
<article>
    <header>
        <h1>文章标题</h1>
        <time datetime="2024-01-15">2024年1月15日</time>
    </header>
    <section>
        <p>内容...</p>
    </section>
    <footer>
        <address>联系方式：email@example.com</address>
    </footer>
</article>

<!-- ❌ 无语义 -->
<div class="article">
    <div class="header">
        <div class="title">标题</div>
        <div class="date">日期</div>
    </div>
</div>
```

## 六、链接优化

### 6.1 内部链接

```html
<!-- ✅ 使用描述性链接文本 -->
<a href="/python-tutorial">Python入门教程</a>

<!-- ❌ 不好的链接文本 -->
<a href="/python-tutorial">点击这里</a>
<a href="/python-tutorial">了解更多</a>
<a href="/python-tutorial">http://example.com/python-tutorial</a>
```

### 6.2 外部链接

```html
<!-- 出站链接可以添加关系 -->
<a href="https://partner-site.com" rel="nofollow">合作伙伴</a>

<!-- 外部链接在新标签页打开 -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
    外部网站
</a>
```

### 6.3 面包屑导航

```html
<nav aria-label="面包屑">
    <ol class="breadcrumb">
        <li><a href="/">首页</a></li>
        <li><a href="/tutorials">教程</a></li>
        <li><a href="/tutorials/python">Python</a></li>
        <li aria-current="page">Python基础</li>
    </ol>
</nav>

<!-- 结构化数据 -->
<nav aria-label="面包屑">
    <ol itemscope itemtype="https://schema.org/BreadcrumbList">
        <li itemprop="itemListElement" itemscope
            itemtype="https://schema.org/ListItem">
            <a itemprop="item" href="/">
                <span itemprop="name">首页</span>
            </a>
            <meta itemprop="position" content="1">
        </li>
        <!-- 更多... -->
    </ol>
</nav>
```

## 七、图片SEO

### 7.1 Alt 属性

```html
<!-- ✅ 描述性alt -->
<img src="python-logo.png" alt="Python编程语言官方Logo">

<!-- ✅ 功能性图片 -->
<a href="/download">
    <img src="download-button.png" alt="下载Python安装包">
</a>

<!-- ✅ 装饰性图片 -->
<img src="decoration.png" alt="">

<!-- ❌ 空alt或无意义描述 -->
<img src="python-logo.png" alt="logo">
<img src="python-logo.png" alt="img01.jpg">
```

### 7.2 图片优化其他要点

```html
<!-- 1. 使用描述性文件名 -->
<!-- ❌ image1.jpg -->
<!-- ✅ python-logo.png -->

<!-- 2. 提供图片尺寸 -->
<img src="photo.jpg" width="800" height="600" alt="..." loading="lazy">

<!-- 3. 使用响应式图片 -->
<img srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 600px) 100vw, 50vw"
     src="photo-800.jpg"
     alt="...">
```

## 八、结构化数据（Schema.org）

### 8.1 常见类型

```html
<!-- 文章 -->
<article itemscope itemtype="https://schema.org/Article">
    <h1 itemprop="headline">文章标题</h1>
    <span itemprop="author">作者名</span>
    <time itemprop="datePublished" datetime="2024-01-15">2024年1月15日</time>
</article>

<!-- 产品 -->
<div itemscope itemtype="https://schema.org/Product">
    <h1 itemprop="name">产品名称</h1>
    <span itemprop="price">¥99.00</span>
    <span itemprop="availability">https://schema.org/InStock</span>
</div>

<!-- 组织 -->
<div itemscope itemtype="https://schema.org/Organization">
    <span itemprop="name">公司名称</span>
    <a itemprop="url" href="https://example.com">官网</a>
</div>

<!-- FAQ -->
<div itemscope itemtype="https://schema.org/FAQPage">
    <div itemscope itemprop="mainEntity" itemtype="https://schema.org/Question">
        <h3 itemprop="name">问题：如何学习Python？</h3>
        <div itemscope itemprop="acceptedAnswer" itemtype="https://schema.org/Answer">
            <span itemprop="text">回答：通过本教程系统学习...</span>
        </div>
    </div>
</div>
```

## 九、URL优化

### 9.1 URL结构

```html
<!-- ✅ 清晰、描述性的URL -->
https://example.com/tutorials/python/python-basics

<!-- ❌ 不好的URL -->
https://example.com/page?id=123
https://example.com/p/123
https://example.com/index.php?cat=5&page=2
```

### 9.2 URL最佳实践

- 使用小写字母
- 用连字符 `-` 分隔单词
- 避免过长的URL
- 使用简单的目录结构
- 保持URL稳定（避免频繁更改）

## 十、技术SEO检查清单

### HTML基础
- [ ] 每个页面有唯一的 `<title>`
- [ ] 有 `<meta description>`
- [ ] 使用语义化HTML标签
- [ ] 正确使用标题层级
- [ ] 图片有 `<alt>` 属性
- [ ] 使用 `<meta charset="UTF-8">`

### 性能
- [ ] 页面加载速度快
- [ ] 移动端适配
- [ ] 使用 HTTPS
- [ ] 避免重复内容

### 技术配置
- [ ] 使用 `<link rel="canonical">` 避免重复内容
- [ ] 有 sitemap.xml
- [ ] 有 robots.txt
- [ ] 实现结构化数据

## 十一、常见SEO错误

```html
<!-- ❌ 错误1：隐藏文字 -->
<div style="display: none;">塞入关键词</div>

<!-- ❌ 错误2：关键词堆砌 -->
<meta name="keywords" content="Python, Python教程, Python学习,
    Python编程, Python入门, Python基础, Python进阶">

<!-- ❌ 错误3：使用框架/Flash -->
<frame src="content.html">
<object data="content.swf">

<!-- ❌ 错误4：过度使用JavaScript渲染内容 -->
<!-- 搜索引擎可能无法抓取JS生成的内容 -->

<!-- ❌ 错误5：缺少面包屑 -->
<!-- 影响搜索引擎理解页面结构 -->
```
