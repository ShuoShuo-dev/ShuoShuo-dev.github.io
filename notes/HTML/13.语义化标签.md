# 语义化标签

## 什么是语义化

语义化是指用合理的HTML标签和属性来描述网页内容的结构，使代码具有良好的可读性，便于开发维护和搜索引擎理解。

## 语义化的重要性

1. **SEO优化**：搜索引擎更好地理解页面内容
2. **可访问性**：屏幕阅读器能正确识别内容
3. **代码可读性**：便于开发人员理解和维护
4. **跨设备兼容**：不同设备能正确解析页面

## 块级语义标签

### header 头部

表示页面或区块的头部区域。

```html
<header>
    <div class="logo">网站Logo</div>
    <nav>导航菜单</nav>
</header>
```

### nav 导航

表示页面的导航链接区域。

```html
<nav>
    <a href="/">首页</a>
    <a href="/about">关于我们</a>
    <a href="/products">产品</a>
    <a href="/contact">联系我们</a>
</nav>
```

### main 主要内容

表示页面的主体内容，一个页面只能有一个。

```html
<main>
    <h1>文章标题</h1>
    <article>
        <p>文章内容...</p>
    </article>
</main>
```

### article 文章

表示独立的文章内容块，如博客文章、新闻、帖子等。

```html
<article>
    <header>
        <h2>文章标题</h2>
        <time datetime="2024-01-01">2024年1月1日</time>
    </header>
    <p>文章正文内容...</p>
    <footer>
        <p>作者：张三</p>
    </footer>
</article>
```

### section 章节

表示文档中的章节/段落，常用于对内容进行分组。

```html
<section>
    <h2>第一章：HTML基础</h2>
    <p>内容...</p>
</section>

<section>
    <h2>第二章：CSS样式</h2>
    <p>内容...</p>
</section>
```

### aside 侧边栏

表示与主内容相关但相对独立的区域，如侧边栏、引用等。

```html
<aside>
    <h3>相关文章</h3>
    <ul>
        <li><a href="#">相关文章1</a></li>
        <li><a href="#">相关文章2</a></li>
    </ul>
</aside>
```

### footer 底部

表示页面或区块的底部区域。

```html
<footer>
    <p>版权信息</p>
    <nav>底部导航</nav>
    <p>联系方式：email@example.com</p>
</footer>
```

## 行内语义标签

### figure 和 figcaption

用于包含图片、图表等及其说明。

```html
<figure>
    <img src="chart.png" alt="销售图表">
    <figcaption>图1：2024年季度销售数据</figcaption>
</figure>
```

### time 时间

表示日期或时间。

```html
<p>发布时间：<time datetime="2024-01-15">2024年1月15日</time></p>
<p>活动时间：<time datetime="2024-01-20T19:00">1月20日晚上7点</time></p>
```

### mark 高亮

用于标记或高亮文本。

```html
<p>关键词：<mark>HTML5</mark> 是最新的HTML标准</p>
```

### progress 进度条

表示任务的完成进度。

```html
<label for="file">下载进度：</label>
<progress id="file" value="65" max="100">65%</progress>
```

### meter 度量

表示在已知范围内的标量值。

```html
<p>磁盘使用：<meter value="50" min="0" max="100">50GB/100GB</meter></p>
<p>评分：<meter value="4.5" min="0" max="5">4.5/5</meter></p>
```

### address 地址

表示联系信息。

```html
<address>
    联系人：张三<br>
    电话：<a href="tel:13800138000">13800138000</a><br>
    邮箱：<a href="mailto:zhangsan@example.com">zhangsan@example.com</a>
</address>
```

## 语义化页面布局示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <header>
        <h1>我的技术博客</h1>

            <a href="/">首页</a>
            <a href="/html">HTML</a>

        <article>
            <header>
                <h2>HTML5语义化标签详解</h2>
                <p>发布时间：<time datetime="2024-01-15">2024年1月15日</time></p>
            </header>

            <section>
                <h3>什么是语义化</h3>
                <p>语义化是指用<mark>合理的HTML标签</mark>来描述网页内容...</p>
            </section>

            <section>
                <h3>语义化标签</h3>

            <figure>

            </figure>

            <footer>
                <p>作者：张三 | 阅读量：<meter value="1520" min="0" max="10000">1520</meter></p>

        </article>

        <aside>
            <h3>相关推荐</h3>

                <li><a href="#">CSS3新特性</a></li>
                <li><a href="#">JavaScript入门</a></li>
                <li><a href="#">Vue3教程</a></li>

        <address>
            <p>联系邮箱：blog@example.com</p>

```

## 语义化vs非语义化

```html
<!-- 非语义化写法 -->
<div class="header">
    <div class="nav">导航</div>
</div>
<div class="main">
    <div class="content">
        <div class="article">内容</div>
    </div>
    <div class="sidebar">侧边栏</div>
</div>
<div class="footer">底部</div>

<!-- 语义化写法 -->
<header>
    <nav>导航</nav>
</header>
<main>
    <article>内容</article>
    <aside>侧边栏</aside>
</main>
<footer>底部</footer>
```

## 总结

| 标签 | 语义 |
|------|------|
| `<header>` | 头部区域 |
| `<nav>` | 导航区域 |
| `<main>` | 主要内容（唯一） |
| `<article>` | 独立文章 |
| `<section>` | 章节/段落 |
| `<aside>` | 侧边栏 |
| `<footer>` | 底部区域 |
| `<figure>` | 插图/图表 |
| `<figcaption>` | 插图说明 |
| `<time>` | 日期/时间 |
| `<mark>` | 高亮文本 |
| `<progress>` | 进度条 |
| `<meter>` | 度量 |
| `<address>` | 联系信息 |

语义化是现代Web开发的重要原则，能提升SEO、可访问性和代码质量。
