# 响应式图片

## 一、为什么需要响应式图片

不同设备的屏幕尺寸和分辨率差异很大：
- 手机：375px 宽度，2x 像素密度
- 平板：768px 宽度
- 桌面：1920px 宽度

如果所有设备都加载同一张大图，会造成：
- 小屏幕设备加载不必要的大图，浪费流量
- 高分辨率屏幕显示模糊
- 页面加载变慢

---

## 二、srcset 属性

### 2.1 基本语法

```html
<img src="small.jpg"
     srcset="small.jpg 480w,
             medium.jpg 800w,
             large.jpg 1200w"
     alt="响应式图片示例">
```

`480w`、`800w` 表示图片的**实际宽度**，浏览器会自动选择合适的图片。

### 2.2 结合 sizes 属性

```html
<img src="small.jpg"
     srcset="small.jpg 480w,
             medium.jpg 800w,
             large.jpg 1200w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 900px) 50vw,
            33vw"
     alt="响应式图片">
```

`sizes` 告诉浏览器：
- 在最大600px的屏幕上，图片占100%视口宽度
- 在最大900px的屏幕上，图片占50%视口宽度
- 其他情况，图片占33%视口宽度

### 2.3 使用像素密度描述符

```html
<!-- 像素密度描述符 -->
<img src="avatar-1x.jpg"
     srcset="avatar-1x.jpg 1x,
             avatar-2x.jpg 2x,
             avatar-3x.jpg 3x"
     alt="用户头像"
     width="200"
     height="200">
```

:::info
像素密度描述符（1x, 2x, 3x）假设图片在所有设备上的显示尺寸相同。
:::

## 三、picture 元素

`picture` 元素让我们可以更灵活地控制图片来源。

### 3.1 基础语法

```html
<picture>
    <source srcset="large.webp" type="image/webp">
    <source srcset="large.jpg" type="image/jpeg">
    <img src="fallback.jpg" alt="图片描述">
</picture>
```

浏览器会从上到下检查：
1. 如果支持 WebP 且条件匹配，加载 WebP 图片
2. 否则继续检查下一个 source
3. 最后使用 img 的 src 作为兜底

### 3.2 根据视口宽度选择图片

```html
<picture>
    <!-- 移动端 -->
    <source media="(max-width: 600px)"
            srcset="mobile-small.jpg 400w,
                    mobile-large.jpg 800w"
            sizes="100vw">

    <!-- 平板 -->
    <source media="(max-width: 900px)"
            srcset="tablet-small.jpg 600w,
                    tablet-large.jpg 1200w"
            sizes="50vw">

    <!-- 桌面 -->
    <img src="desktop.jpg"
         srcset="desktop-small.jpg 800w,
                 desktop-large.jpg 1600w"
         sizes="33vw"
         alt="响应式图片">
</picture>
```

### 3.3 根据屏幕像素密度选择

```html
<picture>
    <source media="(min-width: 800px)"
            srcset="logo-200.png 2x,
                    logo-100.png 1x">

    <img src="logo-50.png"
         srcset="logo-100.png 2x,
                 logo-50.png 1x"
         alt="网站Logo">
</picture>
```

### 3.4 使用现代图片格式

```html
<picture>
    <!-- WebP：更小体积，支持情况良好 -->
    <source srcset="image.webp" type="image/webp">

    <!-- JPEG XR：IE/Edge专用 -->
    <source srcset="image.jxr" type="image/jxr">

    <!-- HEIF：Safari专用 -->
    <source srcset="image.heif" type="image/heif">

    <!-- 兜底：传统JPEG -->
    <img src="image.jpg" alt="图片描述">
</picture>
```

## 四、picture vs srcset

| 特性 | srcset/sizes | picture |
|------|--------------|---------|
| 根据宽度选择 | ✅ | ✅ |
| 根据像素密度选择 | ✅ | ✅ |
| 根据屏幕尺寸选择 | ✅ | ✅ |
| 使用不同图片格式 | ❌ | ✅ |
| 根据浏览器支持选择 | ❌ | ✅ |
| 语法复杂度 | 简单 | 较复杂 |

## 五、实际应用场景

### 5.1 Hero 大图

```html
<!-- 大型横幅图片 -->
<picture>
    <source media="(max-width: 768px)"
            srcset="hero-mobile.jpg"
            type="image/jpeg">
    <source media="(min-width: 769px)"
            srcset="hero-desktop.jpg"
            type="image/jpeg">
    <img src="hero-mobile.jpg"
         alt="网站横幅"
         style="width: 100%; height: auto;">
</picture>
```

### 5.2 用户头像

```html
<!-- 头像通常不需要太高的分辨率 -->
<img src="avatar-200.jpg"
     srcset="avatar-100.jpg 1x,
             avatar-200.jpg 2x,
             avatar-400.jpg 3x"
     alt="用户头像"
     width="100"
     height="100">
```

### 5.3 产品图片

```html
<!-- 产品列表和详情页使用不同尺寸 -->
<picture>
    <source media="(max-width: 600px)"
            srcset="product-list-300.webp 300w,
                    product-list-600.webp 600w"
            sizes="50vw"
            type="image/webp">

    <source media="(max-width: 900px)"
            srcset="product-list-400.webp 400w,
                    product-list-800.webp 800w"
            sizes="33vw"
            type="image/webp">

    <img src="product-desktop.jpg"
         srcset="product-desktop-400.webp 400w,
                 product-desktop-800.webp 800w,
                 product-desktop-1200.webp 1200w"
         sizes="25vw"
         alt="产品图片">
</picture>
```

## 六、图片格式对比

| 格式 | 浏览器支持 | 优点 | 适用场景 |
|------|-----------|------|----------|
| JPEG | 所有 | 兼容性好，体积小 | 照片 |
| PNG | 所有 | 支持透明，质量好 | 图标、Logo |
| GIF | 所有 | 支持动画 | 简单动画 |
| WebP | Chrome/Firefox/Edge | 体积小，质量好 | 现代浏览器 |
| AVIF | Chrome/Firefox | 比WebP更小 | 现代浏览器 |
| SVG | 所有现代浏览器 | 矢量，可缩放 | 图标、Logo、插图 |

## 七、性能优化技巧

### 7.1 使用 CDN

```html
<img src="https://cdn.example.com/image.jpg"
     srcset="https://cdn.example.com/image-400.jpg 400w,
             https://cdn.example.com/image-800.jpg 800w,
             https://cdn.example.com/image-1200.jpg 1200w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="优化后的图片">
```

### 7.2 图片预加载

```html
<!-- 预加载关键图片 -->
<link rel="preload" as="image"
      href="hero.jpg"
      imagesrcset="hero-400.jpg 400w,
                   hero-800.jpg 800w,
                   hero-1200.jpg 1200w"
      imagesizes="100vw">
```

### 7.3 懒加载

```html
<!-- 原生懒加载 -->
<img src="image.jpg"
     srcset="image-400.jpg 400w,
             image-800.jpg 800w"
     loading="lazy"
     alt="懒加载图片">

<!-- 首屏图片不使用懒加载 -->
<img src="hero.jpg" loading="eager" alt="首屏大图">
```
