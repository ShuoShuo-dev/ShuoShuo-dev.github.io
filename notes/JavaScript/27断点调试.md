## 为什么需要断点调试？

代码运行太快，看不清中间过程。断点可以**暂停代码**，逐步查看执行情况

## 打开开发者工具

```
F12 → Source（来源）→ 选择对应的 HTML 文件
```

## 断点操作

| 操作 | 说明 |
|------|------|
| 点击行号 | 添加/移除断点 |
| 刷新页面 | 开始执行，遇到断点暂停 |

## 调试面板介绍

```
┌─────────────────────────────────┐
│  Watch（监视）    │ 作用域      │
│  ├─ num          │ 局部变量     │
│  └─ i           │ 全局变量     │
├─────────────────┴───────────────│
│  Call Stack（调用栈）│ 断点列表  │
├─────────────────────────────────┤
│         代码区域（红色=断点）     │
│    10:  let i = 1;      ●      │
│    11:  while(i <= 3) {        │
└─────────────────────────────────┘
```

## 调试按钮

| 按钮 | 快捷键 | 作用 |
|------|--------|------|
| Step Over | F10 | 跳过（不进入函数） |
| Step Into | F11 | 进入（进入函数内部） |
| Step Out | Shift+F11 | 跳出（跳出当前函数） |
| Resume | F8 | 继续执行到下一个断点 |

## 调试技巧

### 1. 监视变量
```
Watch 面板 → 点击 + → 输入变量名 → 回车
```

### 2. 观察作用域
```
Scope 面板自动显示当前作用域的变量
```

### 3. 条件断点
```
右键断点 → Edit breakpoint → 输入条件（如 i === 3）
```

## 调试示例

```javascript
let sum = 0;
for (let i = 1; i <= 5; i++) {
    sum += i;  // ← 在这行打断点，观察 sum 和 i 的变化
}
console.log(sum);
```

**期望观察**：
- i 从 1 增加到 5
- sum 依次变为 1, 3, 6, 10, 15
