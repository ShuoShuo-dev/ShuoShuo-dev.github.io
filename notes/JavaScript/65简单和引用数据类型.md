# 简单数据类型 vs 引用数据类型

## 分类

| 类型 | 又称 | 包含 | 存储内容 |
|------|------|------|----------|
| **简单类型** | 值类型 / 基本类型 | `string` `number` `boolean` `undefined` `null` | **值本身** |
| **引用类型** | 复杂类型 | 对象、数组、函数 | **地址（指针）** |

## 内存存储机制

```
栈（Stack）              堆（Heap）
┌──────────────┐        ┌──────────────┐
│ let a = 10   │        │              │
│   → 存的是值  │        │  { age: 18 } │
├──────────────┤        │              │
│ let obj = 地址─┼──────→│              │
└──────────────┘        └──────────────┘
```

- **简单类型**：数据存在**栈**里，直接存值
- **引用类型**：栈里存**地址**，真正的数据在**堆**里

## 赋值差异

### 简单类型：互不影响 ✅

```javascript
let a = 10;
let b = a;   // 复制值
b = 20;
console.log(a); // 10，不受影响
```

```
栈：a = 10    栈：a = 10    栈：a = 10
    b = 10        b = 10        b = 20
```

### 引用类型：互相影响 ⚠️

```javascript
let obj1 = { age: 18 };
let obj2 = obj1;   // 复制地址，指向同一个对象！
obj2.age = 20;
console.log(obj1.age); // 20，被影响了！
```

```
栈：obj1 = 0x001 ──→  堆：{ age: 18 }  →  { age: 20 }
    obj2 = 0x001 ──→
```

## 常见坑

```javascript
// 数组也是引用类型
let arr1 = [1, 2, 3];
let arr2 = arr1;
arr2.push(4);
console.log(arr1); // [1, 2, 3, 4]  ← 也被改了！
```

## null 的特殊性

```javascript
console.log(typeof null); // 'object'（历史遗留bug）
```

> `null` 是基本数据类型，但 `typeof` 返回 `'object'`。用途：声明变量准备存对象但对象还没准备好时，先用 `null` 占位。

```javascript
let obj = null;  // 将来要存对象，先用null占位
obj = { name: '张三' };  // 后续再赋值
```
