# 事件流、捕获、冒泡

## 什么是事件流

> 事件完整执行的**流动路径**（从页面到目标元素再回到页面）

```
事件流三阶段：
document → html → body → div → span  （捕获阶段 ↓）
span → div → body → html → document  （冒泡阶段 ↑）
```

## 三个阶段

| 阶段 | 方向 | 说明 |
|------|------|------|
| **捕获** | 从外到内 | `document` → ... → 目标元素 |
| **目标** | 到达目标 | 在目标元素上触发 |
| **冒泡** | 从内到外 | 目标元素 → ... → `document` |

```html
<div class="father">
  <div class="son"></div>
</div>
```

```
点击 .son 时的事件流：

捕获阶段：document → html → body → .father → .son
冒泡阶段：.son → .father → body → html → document
```

## 事件冒泡（重点）

> **默认情况下，事件在冒泡阶段处理**（从内到外）

```javascript
const father = document.querySelector('.father');
const son = document.querySelector('.son');

// 冒泡阶段（默认，第三个参数 false 或不写）
son.addEventListener('click', function() {
  console.log('son 触发');    // 先执行
});
father.addEventListener('click', function() {
  console.log('father 触发'); // 后执行（因为冒泡上来了）
});

// 点击 son → 输出：son 触发 → father 触发
```

## 事件捕获

```javascript
// 第三个参数 true → 捕获阶段执行
father.addEventListener('click', function() {
  console.log('father 捕获'); // 先执行（从外到内）
}, true);
son.addEventListener('click', function() {
  console.log('son 捕获');    // 后执行
}, true);

// 点击 son → 输出：father 捕获 → son 捕获
```

## 阻止冒泡（重要）

> 有时不想让事件传给父元素，需要**阻止冒泡**

```javascript
son.addEventListener('click', function(e) {
  e.stopPropagation();  // 阻止事件传播
  console.log('只有 son 触发');
});

// 点击 son → 只输出：只有 son 触发（father 不会触发了）
```

### 实际场景

```javascript
// 下拉菜单：点击菜单项不应该关闭菜单
menu.addEventListener('click', function(e) {
  e.stopPropagation();  // 阻止冒泡到 document
});

document.addEventListener('click', function() {
  menu.style.display = 'none';  // 点击页面其他地方关闭菜单
});
```

## 小结

| 要点 | 说明 |
|------|------|
| 默认行为 | 冒泡阶段处理（从内到外） |
| 捕获阶段 | 第三个参数 `true` |
| 阻止冒泡 | `e.stopPropagation()` |
| 实际开发 | 冒泡用得多，捕获较少用 |
