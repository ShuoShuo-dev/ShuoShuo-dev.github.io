# JavaScript书写位置

## 一、三种书写位置

| 位置 | 说明 | 推荐度 |
|------|------|:------:|
| 外部JS | 单独的 .js 文件，用 script 引入 | ⭐⭐⭐ |
| 内部JS | 写在 HTML 文件的 script 标签中 | ⭐⭐ |
| 行内JS | 写在 HTML 标签的事件属性中 | ⭐ |

---

## 二、外部JS（推荐）

### 写法

HTML 文件中：
```html
<script src="demo.js"></script>
```

单独的 JS 文件（demo.js）：
```javascript
alert('你好，JS');
```

### 注意

- `<script>` 标签中间**不要写代码**，写了也不会执行
- 可以写多个 `<script src="xxx.js"></script>`

### 优点

- JS代码与HTML分离，结构清晰
- 多个页面可共用同一个JS文件
- 浏览器会缓存JS文件，加载更快

---

## 三、内部JS

### 写法

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>内部JS</title>
</head>
<body>
  <button>按钮</button>

  <!-- JS写在 body 底部 -->
  <script>
    alert('你好，JS');
  </script>
</body>
</html>
```

### 为什么写在 body 底部？

页面从上往下加载，JS写在底部可以确保：
1. 先加载完所有HTML元素
2. 再执行JS代码
3. 避免操作找不到元素

---

## 四、行内JS（了解）

### 写法

```html
<button onclick="alert('点击我')">按钮</button>
```

### 特点

- 直接写在HTML标签里
- 学习阶段了解即可
- Vue框架中会用到这种写法

---

## 五、三种方式对比

| 特性 | 外部JS | 内部JS | 行内JS |
|------|:------:|:------:|:------:|
| 代码分离 | ✅ | ❌ | ❌ |
| 可复用 | ✅ | ❌ | ❌ |
| 浏览器缓存 | ✅ | ❌ | ❌ |
| 维护方便 | ✅ | 一般 | 差 |

---

## 六、实战选择

```
项目规模
├── 小型页面 / 单页面     → 内部JS
├── 中大型项目 / 多页面   → 外部JS
└── Vue/React框架开发     → 外部JS（模块化）
```
