# 遍历对象

对象**没有 length，没有下标**，不能用 `for` 循环和 `forEach`，只能用 `for...in`。

## 语法

```javascript
let obj = { name: '张三', age: 18, gender: '男' };

for (let key in obj) {
  // key 是属性名（字符串类型）
  // obj[key] 是属性值
  console.log(key + ': ' + obj[key]);
}
// 输出：name: 张三  age: 18  gender: 男
```

## ⚠️ 易错点

```javascript
// ❌ 错误：obj.key 相当于 obj['key']，找不到属性
for (let key in obj) {
  console.log(obj.key); // undefined
}

// ✅ 正确：用中括号语法，key 是变量
for (let key in obj) {
  console.log(obj[key]); // 张三, 18, 男
}
```

> **为什么用 `[]` 不用 `.`？**
> `obj.key` 会去找名为 `"key"` 的属性，而对象里没有叫 key 的属性。
> `obj[key]` 会把变量 key 的值代入，相当于 `obj['name']`、`obj['age']`。

## 对象 vs 数组遍历对比

| 对比项 | 数组 | 对象 |
|--------|------|------|
| 遍历方式 | `for` / `forEach` / `for...in` | **只能** `for...in` |
| `key` 的含义 | 下标（索引号） | 属性名 |
| `key` 的类型 | 字符串 | 字符串 |
| 推荐度 | `for...in` 遍历数组**不推荐** | `for...in` 是**唯一选择** |

## 实战：渲染数据到页面

```javascript
let student = { name: '张三', age: 18, hometown: '北京' };

let str = '';
for (let key in student) {
  str += `<p>${key}：${student[key]}</p>`;
}

document.querySelector('.box').innerHTML = str;
// 渲染结果：name：张三  age：18  hometown：北京
```
