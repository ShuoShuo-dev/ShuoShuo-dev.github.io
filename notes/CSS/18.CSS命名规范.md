# 18.CSS命名规范

## 为什么要命名规范？

- 提高代码可读性
- 便于团队协作
- 减少样式冲突
- 便于维护和扩展

## 常用命名规范

### 1. BEM 规范

BEM = Block（块）__ Element（元素）-- Modifier（修饰符）

```css
/* Block */
.card { }

/* Element（属于Block的子元素） */
.card__header { }
.card__body { }
.card__footer { }

/* Modifier（Block或Element的变体） */
.card--featured { }
.card__header--highlighted { }
.card--disabled { }
```

**示例：**
```html
<div class="card card--featured">
  <div class="card__header">
    <h3 class="card__title">标题</h3>
  </div>
  <div class="card__body">
    <p class="card__text">内容</p>
  </div>
  <div class="card__footer card__footer--dark">
    <button class="card__button">按钮</button>
  </div>
</div>
```

### 2. OOCSS 规范

OOCSS = Object-Oriented CSS（面向对象的CSS）

**原则1：结构与皮肤分离**
```css
/* 结构 */
.box { width: 300px; height: 200px; }

/* 皮肤 */
.skin { background: #3498db; border: 1px solid #ddd; }

/* 组合 */
.profile-box {
  @extend .box;
  @extend .skin;
}
```

**原则2：容器与内容分离**
```css
/* 不好 */
.sidebar h3 { color: #333; }

/* 好 */
.sidebar-title { color: #333; }
```

### 3. SMACSS 规范

SMACSS = Scalable and Modular Architecture for CSS

```css
/* Base - 基础样式 */
body { margin: 0; }
a { color: #3498db; }

/* Layout - 布局样式 */
.l-header { }
.l-container { }
.l-sidebar { }
.l-main { }

/* Module - 模块样式 */
.module { }
.module-title { }
.module-content { }

/* State - 状态样式 */
.is-active { }
.is-hidden { }
.is-disabled { }

/* Theme - 主题样式 */
.theme-dark { }
.theme-light { }
```

### 4. Utility-First（功能优先）

```css
/* 工具类 */
.flex { display: flex; }
.grid { display: grid; }
.text-center { text-align: center; }
.text-red { color: red; }
.m-10 { margin: 10px; }
.p-20 { padding: 20px; }
```

### 5. 语义化命名

```css
/* 好 */
.nav { }
.header { }
.footer { }
.sidebar { }
.content { }

/* 不好 */
.left { }
.right { }
.big { }
.red { }
```

## 常见问题

### 1. 选择器过长
```css
/* 不好 */
.nav .nav-list .nav-list-item .nav-list-link { }

/* 好 */
.nav__item-link { }
```

### 2. 选择器过短
```css
/* 不好 */
.a { }
.b { }
.c { }

/* 好 */
.article-title { }
.sidebar-link { }
```

### 3. 过于具体
```css
/* 不好 */
#header .nav ul li a { }

/* 好 */
.nav__link { }
```

### 4. 语义不明确
```css
/* 不好 */
.blue { }
.big { }
.round { }

/* 好 */
.primary-text { }
.heading-size { }
.rounded-border { }
```

## CSS方法论对比

| 方法 | 特点 | 适用场景 |
|------|------|----------|
| BEM | 清晰的结构 | 大型项目，团队协作 |
| OOCSS | 复用性高 | 需要大量复用样式的项目 |
| SMACSS | 模块化 | 中大型项目 |
| Utility-First | 快速开发 | 小型项目，快速原型 |
